# 공용 태그 형식

코드 주석이나 문서 본문에 특정 작업자(인간·AI)를 향한 메모를 남길 때의 형식.

## 형식

```
[to:@수신자 from:@발신자: 코멘트]
```

- `to:`와 `from:` 중 최소 하나는 반드시 포함한다.
- 작업자를 지칭할 때는 앞에 반드시 `@`를 붙인다.
- 파일 형식에 맞는 주석 구문 안에 넣는다 (`//`, `#`, `<!-- -->` 등).

## 성격

이 태그는 AI 전용이 아니다. 인간을 포함한 모든 작업자가 읽고 판단한다. "누구에게 보내는 메시지인가"를 드러내는 수신자 기반 표기이며, 이전에 쓰이던 작성자 기반 태그(`<!-- @이름: ... -->`)를 대체한다.

## 예시

코드 주석:

```kotlin
// [to:@AGENT: 확정된 추상화 레이어]
fun someMethod() { ... }
```

```python
# [to:@AGENT from:@human: 여기 성능 이슈 있음. 벤치마크 필요.]
def process(data): ...
```

문서 본문:

```markdown
<!-- [to:@AGENT: 이 섹션은 예시일 뿐이다. 실제 작성 시 교체.] -->
```

## 해석 원칙

- 태그가 달린 영역은 태그 내용을 하나의 프롬프트와 동등하게 취급하여 먼저 읽고 반영한다.
- 태그가 없다고 해당 영역을 임의로 수정해도 된다는 뜻은 아니다. 기본 판단 원칙은 AGENTS.md의 다른 규정을 따른다.
