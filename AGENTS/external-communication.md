# 외부 협업 소통 규약

AI가 사용자의 토큰, 계정으로 외부 협업 시스템(GitHub PR 본문, 이슈, 댓글, 리뷰 reply 등)에 글을 쓸 때의 공통 소통 규약. PR 맥락의 조립 절차는 [pr-workflow.md](pr-workflow.md)와 그 하위 문서에 두고, 본 규약은 PR 밖(이슈, 일반 댓글, 그 외 외부로 나가는 협업 채널)에도 적용된다.

적용 범위 밖: 채팅 응답, 세션 내 의사 소통, 로컬 명령 출력. 그 경로는 발화자가 AI임이 1:1 대화 맥락에서 자명해 본 규약이 정보를 추가하지 않는다.

## 명시적 멘션 의무

외부 협업 시스템에서 누군가의 코멘트, 스레드에 응답할 때 본인이 글을 단 사실만으로 상대가 알림을 받으리라고 가정하지 않는다. 응답 본문에 듣는 대상을 `@username`으로 명시적으로 멘션한다 - 자동 리뷰 봇도 동일 적용(예: `@gemini-code-assist`). 멘션이 없으면 알림 채널이 닿지 않거나, 닿더라도 누구에게 답하는 글인지 모호해 스레드 가독성이 떨어진다.

복수 대상 멘션은 첫 줄에 모아둔다 - 본 글이 누구를 향한 것인지를 본문 진입과 동시에 보이게.

## AI 시그니처 의무

AI가 사용자의 토큰, 계정으로 외부 협업 시스템에 글을 쓰면 그 글이 사용자의 직접 행동인지 AI 자동 행동인지를 외부 관찰자가 식별할 수 없다. 시그니처가 그 식별 단서가 된다.

형식:

```
Generated with {AI 이름}
```

예: `Generated with Claude Code`, `Generated with Gemini Code Assist`.

도구별 적용:

- PR 본문: 일부 AI(Claude Code 등)는 자동으로 footer에 추가한다. 자동 추가 여부를 확인하고, 없으면 명시 추가.
- 댓글, 리뷰 코멘트 reply: 자동 추가 없는 경우가 많아 명시 의무. 본문과 한 줄 띄운 뒤 마지막에 단다.
