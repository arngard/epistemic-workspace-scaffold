# 원본 보호 원칙

에이전트가 원본(데이터, 콘텐츠)을 다룰 때의 안전 기준. 가역성이 1순위, 공유 기준선 보호가 2순위다. 둘은 독립적으로 작동하며, 변경이 둘 중 하나라도 게이트에 걸리면 사용자 확인을 거친다.

1. 가역성 우선 (1순위): 되돌릴 수 없는 손실은 공유 여부와 무관하게 가장 먼저 막는다. 소스 관리(원격 저장소, 커밋 이력)로 원본을 복원할 수 있는 영역은 에이전트가 직접 작업한다. 복원 수단이 없어 로컬의 그 영역 자체가 유일한 원본일 때만 직접 변경을 피하고 사용자 확인을 거친다. 판단 축은 특정 행위(머지 등)가 아니라 복원 가능성이다. 비가역 손실은 복구 비용이 사실상 무한대이므로 최우선으로 둔다.

2. 공유 기준선 보호 (2순위): 가역성이 확보된(복원 가능한) 범위 안에서 추가로 적용된다. 여러 주체(다른 세션, 협업자, 자동 트리거)가 검토된 상태로 신뢰하는 공유 기준선(`main` 브랜치, `_docs/` SSOT 등)에 변경을 반영할 때는 사용자 검토를 거친다. 복원 가능성과 별개로, 검토되지 않은 상태가 기준선이 되는 것을 막는 신뢰 게이트다. `main` 직접 커밋 금지, 사용자 검토 통과 후 머지(자율 머지 회피), `_docs/` 사람 확인 없는 대폭 재편 금지가 모두 이 원리의 사례다(AGENTS.md "비협상 게이트" 절).

판단 순서: 먼저 복원 가능성을 본다. 복원 불가능하면 공유 여부와 무관하게 사용자 게이트(가역성)를 적용한다. 복원 가능하면 그 다음 공유 기준선인지로 판단한다(공유면 검토 게이트, 비공유 작업 영역이면 자율). 두 고려가 갈리는 모호한 경계에서는 가역성을 따른다.
