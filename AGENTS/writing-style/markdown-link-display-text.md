# 마크다운 링크 표시텍스트

문서 간 참조 링크 `[표시텍스트](대상)`에서 표시텍스트의 정당한 형태를 정한다. audit 스크립트는 링크 대상의 실존만 검사하고 표시텍스트는 검사 밖이므로, 폐지된 파일명이나 옛 경로가 표시텍스트로 남아도 링크가 깨지지 않은 채 독자를 오도한다 (예: `[_strategy/INDEX.md](../AGENTS.md)`는 대상 `../AGENTS.md`가 실존해 링크는 성립하나, 표시텍스트 `_strategy/INDEX.md`는 존재하지 않는 파일을 가리킨다).

원칙은 실존 기반이다 - 파일명이나 경로 형태의 표시텍스트(공백 없이 `.md`로 끝나거나 `/`를 포함)는 실존하는 파일이나 폴더를 정확히 가리켜야 한다. 파일명 형태는 독자가 실존 파일을 지시하는 것으로 읽으므로, 실물과 어긋나면 없는 파일을 가리키는 오도가 된다. 실존을 가리키는 다음 표기가 정당하다.

- 대상 파일명(basename): `[AGENTS.md](daily/AGENTS.md)`.
- 대상 파일의 루트 기준 완전 경로: `[AGENTS/folder-structure.md](../../AGENTS/folder-structure.md)`. 대상은 상대 경로로 두고 표시텍스트로 워크스페이스 내 절대 위치를 보이는 관례.
- 폴더를 가리키는 링크의 폴더명: `[_knowledge/](../_knowledge/AGENTS.md)` (대상은 그 폴더의 디스크립터).

부당한 것은 실존하지 않는 파일을 지칭하는 표기다.

- `[daily.md](daily/AGENTS.md)`: 실물은 `daily/AGENTS.md`이며 `daily.md` 파일은 없다. `.md`를 붙이면 이미 파일명으로 읽히므로 실존하지 않는 이름을 별칭으로 쓰지 않는다.
- `[_strategy/INDEX.md](../AGENTS.md)`: `_strategy/INDEX.md`는 부재 파일 (폐지된 INDEX 잔재).

표시텍스트를 자유 문구로 커스텀하는 것이 정당한 경우는 둘뿐이다.

- 인라인 문구에 건 링크: 서술 문장의 일부에 링크를 거는 경우 (예: 자세한 절차는 단위 작업 흐름 문서에서 다룬다는 식).
- 대상 문서의 제목(타이틀)을 표시텍스트로 쓰는 경우.

본 규칙은 audit 스크립트가 기계 검사한다 - 파일명 형태의 표시텍스트가 실존 대상을 가리키는지 본다. 자연어 문구와 타이틀은 공백을 포함해 파일명 형태에 걸리지 않으므로 검사에서 자연 제외된다.
