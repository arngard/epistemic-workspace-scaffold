# 온톨로지 색인

> "우리 도메인에 어떤 것들이 존재하고, 어떤 관계를 맺는가" — 도메인 개념 정의.
> 같은 단어를 두 사람(또는 두 문서)이 다른 뜻으로 쓸 수 있는 경우에만 정의한다. 보편적인 용어는 정의하지 않는다.

## 사용 규칙

1. 코드에서 도메인 클래스를 설계하기 전에 이 색인에서 해당 개념의 정의를 확인한다.
2. 정의가 없으면 새로 작성하고 이 색인에 등록한다.
3. 정의를 바꾸면 관련 _architecture/ 및 _strategy/ 문서와의 정합성을 확인한다.
