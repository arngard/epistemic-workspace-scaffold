# 핵심 제약

프로젝트 전반에 걸쳐 적용되는 설계 제약. 개별 상세는 각 링크 문서를 참조한다.

## 0. 설계 원칙 (tiebreaker)

설계 판단이 갈릴 때의 우선 기준:

- 대칭성(symmetry): 같은 역할의 요소는 같은 형식을 갖는다. 레이어 간·플랫폼 간 동일 패턴.
- 일관성(consistency): 예외적 패턴을 최소화한다. 한 번 익힌 패턴이 어디서나 재사용된다.
- 예측 가능성(predictability): 새 요소를 추가할 때 그 위치와 형식이 기존 패턴에서 자동으로 도출된다.
- 강타입(strong typing): 데이터 모델을 엄밀하게 정의하여 컴파일 타임에 오류를 잡는다. `Any`, 타입 소거 래퍼, 느슨한 딕셔너리를 지양하고, 모든 값에 구체적 타입을 부여한다. 구조적으로 올바르게 만들어서 컴파일되면 맞게 돌아가는 구도를 추구한다 (correctness by construction). 테스트(귀납적 검증)는 이 위에 보완적으로 얹는 것이지, 타입이 잡아야 할 오류를 테스트에 위임하지 않는다.

이 원칙은 모든 규약과 구조 결정의 최후 tiebreaker이다.
