# 전략 색인

> "우리가 왜 이렇게 하기로 했는가" — 우리의 결정과 그 근거.
> 외부 사실과 환경 정보는 [_knowledge/](../_knowledge/INDEX.md) 참조.
> 최종 설계 명세는 [_architecture/](../_architecture/INDEX.md) 참조.

## 사용 규칙

1. 읽기: 이 색인만 읽고, 필요한 문서만 연다. 전부 읽지 않는다.
2. 쓰기: 내용에 맞는 기존 문서에 추가하거나, 구체적인 제목의 새 파일을 만들고 이 색인에 등록한다.
3. 분할: 단일 문서가 5,000자를 넘으면 하위 주제로 분할한다.
4. 제목 규칙: 파일명은 내용을 특정할 수 있어야 한다. 포괄적 이름은 금지.
