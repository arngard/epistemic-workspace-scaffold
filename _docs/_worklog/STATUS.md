# 상태 레지스트리

> 감사, 주기 작업의 레지스트리. 현재적 문서이며 항상 최신값으로 덮어쓴다 (시간성 규약의 "히스토리 문서"와 구별된다 - cf. [AGENTS/document-temporality.md](../../AGENTS/document-temporality.md)).
>
> 진행 서사를 일절 두지 않는다. 진행 상태는 [TASK_TREE.md](TASK_TREE.md)가, 완료 이력은 git 이력이 담당한다.
>
> 갱신 규칙:
> - 감사, 주기 작업을 수행할 때마다 해당 행의 "마지막 수행"을 갱신한다. 시각은 `yyyy-MM-dd HH:mm:ss` KST 표기.
> - 자식 워크스페이스는 자기 감사, 주기 작업 행을 아래 표에 추가한다.
> - 전역 브랜치 상태는 두지 않는다 - git에는 부모 브랜치 개념이 없어(커밋 DAG만 존재) 정합 유지가 불가능한 파생 정보다. 브랜치는 TASK_TREE에서 단위 작업의 속성으로 다룬다.

| 이름 | 마지막 수행 | 주기/트리거 |
|------|------------|-------------|
| 형식 감사 (`audit.py`) | (아직 없음) | 새 세션 시작 시, push/PR |
| 에피스테믹 의미 감사 (epistemic-auditor) | (아직 없음) | 마지막 수행 12시간 초과 시, 긴 세션 종료 전, 큰 변경 후 |
