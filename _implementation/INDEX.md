# 구현 프로젝트 색인

> 이 워크스페이스에 포함된 구현 프로젝트 목록. `_implementation/` 하위에 프로젝트마다 한 폴더를 만들고 그 폴더에 실제 소스(또는 서브모듈 등)를 둔다. 한 워크스페이스가 여러 프로젝트를 담을 수 있다 (예: Android/iOS SDK 쌍, 서버·프런트 쌍). 표의 각 행은 `_implementation/` 하위의 한 프로젝트 폴더와 1:1 대응한다.
> 프로젝트 폴더 이름은 가급적 해당 프로젝트의 리모트 레포 이름을 고려하여 짓는다.
> 공통 사용 규칙은 [AGENTS.md](../AGENTS.md)의 "색인(INDEX) 사용 규칙" 섹션 참조.
